# ExampleProject Rules

This sanitized project rule file is an example of `Projects/<name>/RULES.md`, which WorkbenchStateSync includes.

## Commit Drafting

- Read this file before drafting project-specific commit or DevLog text.
- Keep project-specific title formats, DevLog paths, and encoding rules here instead of in the public workbench rules.

## Review Boundaries

- Review against `Projects/ExampleProject/baseline/`.
- Proposed edits belong under `Projects/ExampleProject/edit/<agent>/`.
- Do not sync `baseline/` or `edit/` through WorkbenchStateSync.
